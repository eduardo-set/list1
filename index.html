🖐️ Команда /swing Новое

Новая команда, позволяющая размахивать руками сущностей.
Команда успешно работает в отношении всех сущностей, но не каждая из них поддерживает анимацию размахивания. Клиент будет отображать анимацию только тогда, когда это возможно.
Например, команда позволяет размахивать руками манекена.
Выводит количество сущностей, которые размахивают руками.

Синтаксис: /swing <селектор сущности> <основная рука|вторая рука>

💡 Атрибуты окружения Новое

minecraft:visual/block_light_tint

Цвет освещения блока. Изначально имеет тёмно-серый цвет при низком уровне освещённости, становится окрашенным этим атрибутом при среднем уровне и белым при высоком уровне. По умолчанию он придаёт желтоватый оттенок факелам. Для этого атрибута лучше всего подходят яркие цвета, при этом хотя бы один цветовой компонент должен быть полностью ярким. Оттенок применяется глобально ко всем источникам света, видимым на экране. Отдельные источники света не могут быть окрашены по-разному.
Тип значения: цвет RGB
Значение по умолчанию: #FFD88C
Модификаторы: модификаторы цвета RGB
Интерполяция: да
Разрешено в точке расположения камеры

minecraft:visual/ambient_light_color

Определяет оттенок и яркость окружающего освещения. Этот свет применяется к миру при нулевом уровне освещённости. Сверху добавляется свет от блоков и неба.
Тип значения: цвет RGB
Значения по умолчанию: #0A0A0A (Верхний мир), #302821 (Незер), #3f473f (Энд)
Модификаторы: модификаторы цвета RGB
Интерполяция: да
Разрешено в точке расположения камеры

minecraft:visual/night_vision_color

Этот цвет используется аналогично цвету окружающего освещения. Когда эффект «Ночное зрение» активен, в качестве окружающего цвета используется максимум значений minecraft:visual/night_vision_color и minecraft:visual/ambient_light_color для каждого компонента. «Ночное зрение» по умолчанию не окрашивает свет.
Тип значения: цвет RGB
Значение по умолчанию: #999999
Модификаторы: модификаторы цвета RGB
Интерполяция: да
Разрешено в точке расположения камеры

⏰ Временные метки и мировые часы Новое

Временные метки назначают определённое имя конкретному (опционально повторяющемуся) моменту времени для конкретных мировых часов. Временные метки заменяют ранее предопределённые имена, которые могли использоваться командой /time set, например day или night.

Временные метки существуют в контексте конкретных мировых часов. Это означает, что, хотя разные временные метки могут быть определены разными таймлайнами, для одних мировых часов может существовать только одна временная метка с определённым идентификатором. Также это означает, что у двух разных мировых часов могут быть временные метки с одинаковым именем.

Встроенные применения временных меток

Некоторые временные метки используются для управления поведением немодифицированной игры. Как правило, они настроены так, чтобы не отображаться в командах /time.

  • minecraft:wake_up_from_sleep — время, до которого следует продвинуть стандартные часы измерения, когда игроки в этом измерении просыпаются после сна.
  • minecraft:roll_village_siege — момент времени стандартных часов измерения, в который должна быть вычислена случайная вероятность начала осады деревни.

Мировые часы

Мировые часы содержат внутреннее время, которое увеличивается каждый такт. Они могут быть определены в наборах данных в реестре world_clock (по пути data/<namespace>/world_clock/<id>.json).
Мировые часы могут быть приостановлены или возобновлены, а их время может быть изменено или запрошено с помощью команды /time.
Формат: объект без полей

📊 Наборы сделок Новое

Находятся в папке trade_set и используются деревенскими жителями и странствующими торговцами.
В настоящее время существуют только жёстко закодированные варианты в папках <профессия>/level_<уровень> для деревенских жителей и в папках wandering_trader/buying, wandering_trader/special, wandering_trader/common для странствующих торговцев.

Формат записи:
trades — идентификатор сделки деревенского жителя, список идентификаторов сделок с определённым пространством имён или тег сделки с префиксом хеша, которые определяют сделки, входящие в этот набор.
amount — числовой параметр, представляющий количество сделок, которые будут сгенерированы из этого набора при использовании.
allow_duplicates — логическое значение, указывающее, может ли набор сделок использовать одну и ту же сделку несколько раз для генерации. По умолчанию false.
random_sequence — необязательная именованная случайная последовательность, определяющая, какие сделки будут сгенерированы.

{ "amount": 2.0, "trades": "#minecraft:armorer/level_1", "random_sequence": "minecraft:trade_set/armorer/level_1", "allow_duplicates": true }

🔍 Отладочный рендерер карты освещения Новое

Добавлен новый отладочный рендерер для карты освещения, который можно включить с помощью сочетания клавиш F3 + 4.
Несовместим с графиками FPS/TPS и сетевыми графиками.
Отображает текстуру карты освещения в реальном времени в правом нижнем углу экрана.
Вертикальная ось текстуры представляет уровни небесного освещения (0-15 снизу вверх).
Горизонтальная ось текстуры представляет уровни освещения от блоков (0-15 слева направо).
Цвет каждого пикселя представляет собой результирующий цвет, который будет применён к блокам и сущностям, которые освещаются соответствующими уровнями освещения блоков и неба.

🔢 Поставщики чисел и предикаты Новое

minecraft:environment_attribute

Получает и предоставляет значение атрибута окружения (которое может быть представлено числом). Этот поставщик требует контекста с заданной позицией origin, пока атрибут окружения может изменяться по позиции.
Формат: объект с полями: attribute — идентификатор атрибута окружения для получения.
Пример: {type: 'environment_attribute', attribute: 'gameplay/sky_light_level'}

minecraft:sum

Выводит сумму списка поставщиков чисел.
Поля: summands — список поставщиков чисел.

minecraft:environment_attribute_check

Предикат проверки добычи, точно соответствующий значению атрибута окружения в заданной позиции. Для этого предиката требуется контекст с заданной позицией origin, если атрибут окружения может изменяться в зависимости от позиции.
Формат: объект с полями: attribute — идентификатор атрибута окружения для проверки; value — точное значение атрибута окружения для сопоставления.
Пример: {condition: 'environment_attribute_check', attribute: 'gameplay/piglins_zombify', value: true}

💼 Сделки, управляемые данными Новое

Сделки деревенских жителей и странствующих торговцев теперь управляются данными, благодаря чему могут быть настроены разработчиками наборов данных.
Сделки деревенских жителей находятся в папке villager_trade и представляют собой основу для торговли, которая используется жителями и странствующими торговцами для генерации фактических сделок.

Формат записи:
wants — объект, представляющий тип предмета и количество предметов, необходимых для торговли (id, count, components).
additional_wants — необязательный объект, представляющий дополнительный предмет для продажи торговцу.
gives — объект, представляющий собой итоговый предмет, купленный торговцем.
given_item_modifiers — необязательный список модификаторов, применяемых к предмету в процессе торговли.
max_uses — максимальное количество использований (по умолчанию 4).
reputation_discount — влияние скидок на стоимость (по умолчанию 0.0).
xp — количество опыта торговца (по умолчанию 1).
merchant_predicate — предикат условий для торговца.
double_trade_price_enchantments — идентификаторы чар, удваивающих стоимость.

🧩 Компоненты данных Новое

minecraft:additional_trade_cost

Временный компонент, используемый для изменения необходимого количества предметов, которые можно купить у жителя; удаляется после генерации сделки. Формат: целое число.

minecraft:dye

Определяет цвет красителя. Используется в тех случаях, в которых ранее использовался определённый предмет красителя, включая взаимодействие с мобами и блоками. Формат: один из следующих цветов: white, orange, magenta, light_blue, yellow, lime, pink, gray, light_gray, cyan, purple, blue, brown, green, red, black.

🎲 Функции добычи Новое

Добавлен тип таблицы добычи minecraft:villager_trade, который принимает параметры: this — сущность, предлагающая торговлю; origin — позиция блока у ног сущности.

minecraft:set_random_dyes

Аддитивно добавляет красители к компоненту предмета minecraft:dyed_color, если предмет находится в теге #dyeable. Поля: conditions — условия для применения функции; number_of_dyes — числовой поставщик, представляющий количество случайных красителей из шестнадцати, которые будут применены к предмету.

minecraft:set_random_potion

Случайным образом устанавливает компонент данных minecraft:potion_contents для предмета. Поля: conditions — условия применения функции; options — необязательный идентификатор зелья, список идентификаторов зелий с пространством имён или тег зелья с префиксом хеша, которые представляют возможные зелья для выбора.

🧱 Изменения блоков

Камнерез

Из глубинного сланца на камнерезе теперь можно изготовить дроблёный и полированный варианты, а также кирпичи и плитняк. Из камня на камнерезе теперь можно изготовить булыжник и другие блоки из булыжника.

Музыкальный блок

Добавлен новый инструмент — труба. Можно воспроизвести, поставив музыкальный блок на медный блок. Воспроизводимые звуки меняются в зависимости от стадии окисления: block.note_block.trumpet, block.note_block.trumpet_exposed, block.note_block.trumpet_oxidized, block.note_block.trumpet_weathered.

Растяжка

Текстура растяжки теперь отрисовывается как альфа-канал, а не как прозрачная.

🔧 Изменения предметов

Бирка

Теперь создаётся из 1 бумаги и 1 любого кусочка (золота, железа, меди).
Теперь может быть куплена у странствующего торговца за 1 изумруд с шансом 7%. В рамках одной сделки бирку можно купить 5 раз.
Больше не может быть куплена у библиотекаря уровня «Мастер».
Больше не может быть найдена в сундуках древнего города и лесного особняка.

Копьё

Пороговые значения, при которых копьё может сбросить противника с ездового животного, приведены к значениям из Bedrock Edition.

Мешок

Теперь поддерживает компонент bundle_container с весом больше 2³¹−1, что эквивалентно 2³¹−1 стопкам предметов. Ранее компонент удалялся при превышении лимита. Мешок, превышающий этот лимит, отображается как полный, однако при попытке очистить его, добавить новый предмет или просмотреть всплывающую подсказку он будет отображаться пустым.

Стопка предметов

Теперь во многих случаях считается пустой, если содержит конфликтующие данные, например, если количество предметов превышает max_stack_size или если max_stack_size больше единицы вместе с max_damage. Шалкеровый ящик, содержащий стопку предметов с конфликтующими данными, будет отображать в подсказке «Воздух x0», а при его размещении такая стопка будет удалена. Выстрел стрелой с конфликтующими данными из арбалета не будет засчитан.

Верблюд-кадавр

Удалена модель детёныша.

Детёныши

Добавлены новые звуки для волчат, котят, поросят, жеребят и цыплят.
Обновлены модели и текстуры детёнышей и детей всех мобов, за исключением гастят и детёнышей наутилусов, а также изменены хитбоксы некоторых детёнышей для соответствия их новым моделям.
Седло и броня больше не отображаются на поросятах, верблюжатах и волчатах.
Детёныши лошади-скелета и лошади-зомби теперь могут быть призваны только с помощью яиц призывания или команд.
Дети деревенских жителей и зомби-жителей теперь имеют базовую модель, поверх которой накладываются различные слои.

Коза

Козлёнок теперь вращает головой, когда бодает.

Кролик

Обновлены модель и текстура взрослой особи и детёныша. Текстуры кролика из пакета «Классический» были удалены в связи с обновлением модели.

Курица

Исправлены полосы на тропической курице.

Лошадь

Чубарый окрас взрослой лошади (отметины в виде чёрных пятен) был обновлён, чтобы быть визуально ближе к этому же окрасу у обновлённых жеребят.

Лошадь-зомби и лошадь-скелет

Больше не паникуют при получении урона.

Свинья

Добавлены звуки приёма пищи.

Страйдер

В соответствии с Bedrock Edition цвет маленького страйдера теперь зависит от цвета взрослой особи, на которой он стоит.

Стойка для брони

Маленькая стойка для брони теперь корректно отображает надетую броню.

⌨️ Формат команд

/fetchprofile

Добавлена новая подкоманда, которая будет выводить информацию о профиле сущности в игровом мире.
Синтаксис: /fetchprofile entity <селектор отдельной сущности>
Если у целевой сущности нет профиля (в настоящее время он есть только у игроков и манекенов), эта команда выдаст ошибку.

/time

Команда /time теперь основана на мировых часах. Часы можно дополнительно указать в качестве вводных данных с помощью /time of <clock> .... Если часы не указаны, используются часы, стандартные для данного измерения. Возвращаемое значение из подкоманд set и add теперь представляет собой общее количество прошедших тактов часов вместо текущего времени суток.

/time [of <clock>] set <time> — устанавливает общее количество прошедших тактов часов. /time [of <clock>] set <timemarker> — переводит часы вперёд к следующему появлению временной метки. /time [of <clock>] add <time> — добавляет такты к часам; может быть отрицательным или положительным числом. /time [of <clock>] pause — приостанавливает часы. /time [of <clock>] resume — возобновляет часы. /time [of <clock>] query <timeline> — отображает и возвращает текущее количество тактов в периоде таймлайна. /time [of <clock>] query <timeline> repetitions — отображает и возвращает количество повторений периода таймлайна. /time [of <clock>] query time — отображает и возвращает общее количество прошедших тактов часов. /time query gametime — как и прежде, общее количество прошедших тактов в игровом мире. /time [of <clock>] rate <rate> — устанавливает множитель скорости, с которым течёт время. rate — число с плавающей точкой от 0 (исключая) до 1000 (включительно).

🌍 Изменения мира и level.dat

Тег player заменён на тег singleplayer_uuid, указывающий на используемый файл данных игрока.
Настройки сложности перемещены в тег difficulty_settings.
Значение Difficulty переименовано в difficulty и теперь является строковым: peaceful, easy, normal, hard.
Значение DifficultyLocked переименовано в locked.
Данные для битвы с эндер-драконом перемещены в папку data: dimensions/minecraft/the_end/data/minecraft/ender_dragon_fight.dat.
Битву с эндер-драконом можно активировать и в других измерениях.
Изменения данных битвы с драконом: NeedsStateScanning → needs_state_scanning, DragonKilled → dragon_killed, PreviouslyKilled → previously_killed, IsRespawning → respawn_stage (start, preparing_to_summon_pillars, summoning_pillars, summoning_dragon, end), Dragon → dragon_uuid, ExitPortalLocation → exit_portal_location, Gateways → gateways. Добавлены respawn_time и respawn_crystals.
Данные странствующего торговца: data/minecraft/wandering_trader.dat. Удалено WanderingTraderId, переименованы WanderingTraderSpawnChance → spawn_chance, WanderingTraderSpawnDelay → spawn_delay.
Данные погоды: data/minecraft/weather.dat. Переименованы clearWeatherTime → clear_weather_time, rainTime → rain_time, thunderTime → thunder_time.
Данные игроков перемещены в подкаталог players: advancements, playerdata, stats.
Все стандартные измерения теперь хранятся в подпапке dimensions: dimensions/minecraft/overworld, dimensions/minecraft/the_nether, dimensions/minecraft/the_end.
Пакет ресурсов мира (resources.zip) перемещён в подкаталог resourcepacks.
Данные, сохранённые в папке data, теперь имеют пространство имён.
Файл chunks.dat переименован в chunk_tickets.dat.
Данные для карт теперь хранятся в data/minecraft/maps.
Файл idcounts.dat переименован в last_id.dat.
Особый случай raids_end.dat больше не используется. Энд теперь использует raids.dat.
Структуры, сохранённые структурными блоками, перемещены из generated/namespace/structures в generated/namespace/structure.

💡 Алгоритм карт освещения

Был заменён на новый алгоритм, который в целом должен давать те же результаты, но имеет следующие отличия:
• Он проще и понятнее, благодаря чему был исправлен ряд ошибок предыдущего алгоритма карт освещения.
• Эффект «Тьма» и затемнение неба при битве с иссушителем теперь работают одинаково во всех измерениях.
• Эффект «Ночное зрение» теперь просто добавляет окружающее освещение, а не масштабирует результирующие цвета.
• Это означает, что полностью тёмные области больше не выглядят ярче, чем области, освещённые блоками или небесными телами.
• Составляющие алгоритма, такие как оттенок света от блоков, цвет окружающего освещения и цвета при эффекте «Ночное зрение», теперь управляются данными через атрибуты окружения.

🏔️ Генерация мира

В настройках поставщиков состояния блоков на основе правил произошли изменения: их параметр fallback стал необязательным и может быть пустым.
Декорация forest_rock переименована в block_blob. Конфигурация декорации block_blob изменена для поддержки параметров: state — блок, из которого сделан валун; can_place_on — предикат блока.
Декорация ice_spike переименована в spike. Конфигурация поддерживает: state, can_place_on, can_replace.
Конфигурации декораций huge_red_mushroom и huge_brown_mushroom получили параметр can_place_on.
В конфигурации декорации tree параметры force_dirt и dirt_provider заменены одним поставщиком состояния блока below_trunk_provider.
Поставщики состояния блоков теперь имеют новый тип — rule_based_state_provider. Теперь они могут использоваться в конфигурациях всех декораций.
Декорации, создаваемые из костной муки, больше не ограничиваются типом декорации flower и вместо этого управляются тегом #can_spawn_from_bone_meal.
Типы декораций flower, flower_no_bonemeal и random_patch были удалены.
Добавлен поставщик целых чисел trapezoid с полями: min, max, plateau.
Модификатор размещения count теперь ограничен диапазоном от 0 до 4096, а не до 256.
